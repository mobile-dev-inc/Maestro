THE CORE API = THIS CONTRACT (not YAML)
INPUTS — Command IR
Typed, versioned, cross-platform commands & actions.
OBSERVATION — Perceive primitives
observe(query) / snapshot() / exists(); one typed Element model (role · name · state · bounds · hittable). Cheap targeted queries, not full-tree dumps.
ERRORS — Result values
Success | Failure, categorized by who-fixes-it (Author · Test · App · Infra). Nothing thrown across the boundary.
TRACE — Span stream
run → flow → command → retry → action. Streamable live AND persisted to disk from one source of truth.
▼
CONTROL-PLANE ENGINE (host-side)
Selector resolution (re-resolvable matchers) Actionability + event-driven synchronization (no 'settle' sleeps) Retry / wait / timeout policy Emits the trace span stream

All policy lives here, host-side.

▼
ONE PERSISTENT BIDIRECTIONAL CHANNEL (per device session)
◀ ─────────────────────────────── ▶

Carries commands + subscribable events + observation over one resumable channel (WebDriver-BiDi / CDP style). Per-command deadlines & cancellation — one timeout never kills the channel.

▼
THIN ON-DEVICE AGENT (per platform)
Capture — a11y tree + screenshots
Push events — ui.changed · idle.reached
Primitive input — tap / swipe / type

No matching, waiting, or policy on device → far less brittle, easy to evolve. Pushed events replace today's snapshot-polling.

▼
Android — UiAutomator / a11y
iOS — XCUITest
Optional gray-box idle SDK — opt-in for exact idle signals
THE 5 KEY MOVES
  1. YAML becomes an adapter, not the contract — the typed Command IR is the input.
  2. The contract is four co-equal dimensions: inputs · observation · errors · trace.
  3. Event-driven sync (device pushes 'idle' / 'changed') replaces snapshot-poll-and-diff → fast + reliable.
  4. A thin on-device agent + one resumable channel → robust, evolvable transport.
  5. The batch runner is just one consumer; agents call the perceive-act primitives directly.
